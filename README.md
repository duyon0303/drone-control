# 🚁 오늘의 드론 공부
## 1. Git Conflict
### 1.1 git conflictf란?
#### 두 명 이상의 사람이 같은 파일의 같은 줄을 동시에 수정하고 저장하려고 할 때 발생하는 충돌 현상이다.
### 1.2 왜 발생하는 걸까?
#### 웹에서 파일의 제목을 '항공기 제어'라고 고치고, 동시에 터미널에서도 같은 줄을 '드론 제어'라고 고치고 올리려고 할때, 깃허브 서버는 웹 버전이 맞니, 터미널 버전이 맞니?라며 거절 메시지를 보낸다.
### 1.3 해결하는 방법
#### 우선 깃허브의 내용을 내 컴퓨터로 강제로 가져온다. 이때 깃이 파일 안에 충돌 난 부분을 표시해 줌.
#### 파일 안의 <<<<<<< HEAD 와 >>>>>>> 표시를 찾아서 남기고 싶은 내용만 남기고 나머지는 지워준다.
#### 수정한 파일을 다시 저장해서 올리면 해결됨!
#### ! 프로젝트를 시작하기 전에 항상 git pull origin main 을 먼저 해서 최신 상태를 유지하는 것이 가장 좋은 방법!
## 2. Fork&Clone
### 2.1 Fork
#### 다른 사람의 원격 저장소를 나의 원격 저장소로 복제하는 것을 말한다.
#### 남의 Github 저장소 -> 나의 Github 저장소
#### 원본 프로젝트에 영향을 주지 않고 코드를 수정해볼 수 있다.
### 2.2 Clone
#### 원격 저장소(GitHub)에 있는 코드를 PC로 내려받는 것을 말한다.
#### Git의 기본 명령어(git clone)이다.
#### 직접 코드를 수정하고 컴파일 할 수 있다.
#### 해당 저장소에 권한이 있다면 수정 후 다시 push 할 수 있다.
#### 실제 코딩 작업을 시작하기 위해 프로젝트 파일을 내 컴퓨터로 가져올 때 사용한다.
